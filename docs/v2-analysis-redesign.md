# 데이터 분석 페이지 재설계

## 상태: 기획 진행 중

## 문제 정의

현재 데이터 분석 페이지는 "분석"이 아니라 "데이터 뷰어"에 불과함.
- 필터 → 차트 1개 → 테이블 1개의 단순 구조
- 지역 비교 섹션이 필터와 단절 (항상 고정 데이터)
- 인사이트를 제공하지 않음 — 사용자가 스스로 해석해야 함
- 기본 필터값이 시즌 외에는 빈 데이터를 보여줌

## 목표

원하는 데이터를 원하는 방향으로 쉽게 조합하고, 비교하고, 인사이트를 얻을 수 있는 분석 페이지.

## 사용 가능한 데이터

- **기간**: 2013~2025년 (약 12년)
- **지역**: 강원, 경북, 경남 (3개)
- **조합**: 지역별 종속 (총 ~16개)
- **등급**: 6개 (1등품, 2등품, 3등품생장정지, 3등품개산, 등외품, 혼합품)
- **필드**: 수량(kg), 단가(원/kg), 금일/전일/누계
- **시즌**: 매년 8~12월
- **저장소**: IndexedDB (복합 인덱스: date+region, date+union, date+region+union)

## 사용자가 물어볼 질문들

분석 페이지가 답해야 하는 질문:

1. **이 기간에 시세가 어떻게 변했나?** → 가격 추이 차트
2. **올해와 작년, 뭐가 달랐나?** → 기간 비교 (오버레이 or 변동률)
3. **어느 지역이 가장 비싼가?** → 지역별 가격 비교
4. **어느 등급이 가장 많이 거래됐나?** → 등급별 비중
5. **물량이 많으면 가격이 떨어지나?** → 가격-물량 상관관계
6. **이 필터로 본 핵심 수치는?** → 요약 KPI (평균단가, 총거래량, 최고/최저, 거래일수)
7. **원본 데이터를 직접 보고 싶다** → 상세 테이블

## 페이지 구조

```
┌─────────────────────────────────────────────────────┐
│  필터 바                                              │
│  기간 선택 | 지역 (복수) | 조합 | 등급 (복수)           │
│  [비교 모드 토글] → ON이면 비교 기간 선택 노출           │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  요약 KPI (필터 결과 기반)                              │
│  평균 단가 | 총 거래량 | 최고가 | 최저가 | 거래일수      │
│  비교 모드 ON → 각 KPI에 전기 대비 변동률 표시           │
└─────────────────────────────────────────────────────┘

┌──────────────────────┐  ┌───────────────────────────┐
│  가격 추이             │  │  등급별 비중                │
│  등급별 라인차트        │  │  수량 or 금액 기준          │
│  비교 모드: 오버레이    │  │  수평 바차트               │
│  X: 날짜, Y: 단가      │  │                           │
└──────────────────────┘  └───────────────────────────┘

┌──────────────────────┐  ┌───────────────────────────┐
│  가격-물량 상관         │  │  지역 비교                 │
│  산점도                │  │  필터 기간 내 지역별        │
│  X: 물량, Y: 단가      │  │  평균단가 수평 바차트       │
│  등급별 색상 구분       │  │  필터 연동                 │
└──────────────────────┘  └───────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  상세 테이블                                          │
│  필터 결과 데이터 (정렬 가능, 페이지네이션)              │
└─────────────────────────────────────────────────────┘
```

## 필터 시스템

### 기본 필터
- **기간**: DatePicker 시작일~종료일
- **지역**: 복수 선택 가능 (기존은 단일). 전체 선택 옵션
- **조합**: 선택한 지역에 종속. 복수 선택 가능
- **등급**: 복수 선택 (기존 유지)

### 비교 모드
- 토글로 ON/OFF
- ON이면 "비교 기간" DatePicker 추가 노출
- 가격 추이 차트에 비교 기간 오버레이
- 요약 KPI에 전기 대비 변동률 표시

### 기본값 개선
- 현재: 시즌 외이면 빈 데이터
- 개선: 가장 최근 데이터가 있는 시즌의 마지막 30일로 설정
- daily-manifest.json의 `latestDate` 활용

## 분석 섹션 상세

### 1. 요약 KPI

필터 결과에서 실시간 계산:

| KPI | 계산 방식 |
|-----|----------|
| 평균 단가 | 선택 등급의 가중 평균 (수량 기준) |
| 총 거래량 | 선택 등급 수량 합계 |
| 최고가 | 선택 등급 중 최고 단가 (날짜·등급 표시) |
| 최저가 | 선택 등급 중 최저 단가 (날짜·등급 표시) |
| 거래일수 | 필터 기간 내 데이터가 있는 날짜 수 |

비교 모드 ON이면 각 KPI 아래에 `▲ 12.3%` 또는 `▼ 5.2%` 변동률 표시 (앰버/틸).

### 2. 가격 추이 차트

- 기존 DataAnalysisChart (연도별 서브플롯) 유지하되 개선
- 가격/수량 토글
- 비교 모드: 비교 기간 데이터를 반투명 오버레이로 표시
- 필터의 모든 조건에 반응

### 3. 등급별 비중

- 수평 바차트 (비율 표시)
- 수량 기준 / 금액 기준 토글
- 필터 기간 내 데이터 기반
- 등급 컬러 테마 사용

### 4. 가격-물량 상관

- 산점도 (scatter plot)
- X축: 일일 물량 (kg), Y축: 단가 (원/kg)
- 등급별 색상 구분 (등급 컬러)
- 호버 시 날짜·지역·조합·등급 표시
- 추세선 (선형 회귀) 옵션

### 5. 지역 비교

- 필터 기간 기반 (기존: 고정 manifest 데이터 → 개선: 필터 연동)
- 지역별 평균 단가 수평 바차트
- 복수 지역 선택 시 해당 지역만, 전체 선택 시 3개 지역 모두
- D3.js 바차트

### 6. 상세 테이블

- 기존 테이블 구조 유지
- 정렬 기능 (컬럼 헤더 클릭)
- 페이지네이션

## 기존 코드 활용

### 유지
- `analysisUtils.ts` — 필터 로직, 데이터 변환 (확장 필요)
- `dataAnalysisLoader.ts` — IndexedDB 쿼리
- `DataAnalysisChart.tsx` — 가격 추이 차트 (리팩토링)
- `TableSection.tsx` + `Table/` — 상세 테이블
- `database.ts` — IndexedDB 스키마, 복합 인덱스

### 교체
- `AnalysisFilters.tsx` — 복수 지역 선택 + 비교 모드 추가
- `ChartSection.tsx` — 단일 차트 래퍼 → 불필요 (직접 배치)
- `RegionComparisonSection.tsx` — 고정 데이터 → 필터 연동

### 신규
- 요약 KPI 컴포넌트
- 등급별 비중 차트 컴포넌트
- 가격-물량 산점도 컴포넌트
- 비교 모드 KPI 변동률 계산 유틸

## 데이터 흐름

```
필터 변경
  ↓
IndexedDB 쿼리 (loadDateRangeData)
  ↓
filteredData (MushroomAuctionDataRaw[])
  ↓
┌────────────┬─────────────┬──────────────┬──────────────┐
│ 요약 KPI    │ 추이 차트    │ 비중 차트     │ 산점도       │
│ (useMemo)  │ (useMemo)   │ (useMemo)    │ (useMemo)    │
└────────────┴─────────────┴──────────────┴──────────────┘

비교 모드 ON
  ↓
비교 기간 IndexedDB 쿼리
  ↓
comparisonData (MushroomAuctionDataRaw[])
  ↓
┌────────────┬─────────────┐
│ KPI 변동률  │ 추이 오버레이 │
└────────────┴─────────────┘
```

모든 분석 섹션은 동일한 filteredData를 소비. 각 섹션은 useMemo로 필요한 형태로 변환.

## 결정 사항

- [x] 복수 지역 선택 시 조합 필터: **합집합** — 선택된 지역들의 모든 조합 표시, 미선택 시 전체 데이터
- [x] 비교 모드 비교 대상: **작년 동기간 자동 설정** — 선택 기간 -1년, 수동 변경 가능
- [x] 산점도 추세선: **기본 OFF, 토글로 ON** — 데이터 적을 때 오해 방지
- [x] 모바일 레이아웃: **1열 스택** — 탭 방식은 차트 간 비교 불가, 스크롤이 길어도 흐름 파악 우선
