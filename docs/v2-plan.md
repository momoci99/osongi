# 오송이 V2 기획

## 상태: 기획 완료, 구현 대기

## 배경

- V1: React 19 + MUI v7 + D3.js SPA
- shadcn/ui 전환 시도했으나 전반적인 UI 완성도 불만족으로 롤백
- V2에서는 UI 전환 + 기획 재정비 + 지표 강화를 함께 진행

## 핵심 방향

1. **UI 라이브러리**: MUI v7 유지 (V1 기반 위에 점진적 기능 추가)
2. **차트 라이브러리**: D3.js 유지 (개인 선호)
3. **기획 재정비**: 시즌 특성을 반영한 대시보드 + 유의미한 지표 추가

> shadcn/ui + Tailwind 전환은 한번 시도 후 롤백한 경험이 있고,
> V1의 D3 차트·데이터 로더·IndexedDB가 이미 잘 동작하므로
> 클린 리라이트 대신 MUI v7 위에서 기능을 추가하는 방향으로 결정.

## 결정 사항

### 모바일 퍼스트

스마트폰에서 문제없이 동작해야 함. 송이버섯 시세를 현장(산지, 시장)에서 확인하는 사용 시나리오를 고려.

- 모든 페이지·차트·테이블이 모바일에서 사용 가능해야 함
- D3 차트: 터치 인터랙션, 가로 스크롤/줌, 적절한 최소 크기 보장
- 테이블: 가로 스크롤 또는 카드형 레이아웃 전환
- 네비게이션: 모바일 햄버거/바텀 네비 검토

### 사용자 시나리오

주요 사용자:
1. **지표 확인 사용자** (PC 웹): 송이버섯 시장 전체 동향·통계를 분석하고 싶은 사람
2. **현장 시세 확인 사용자** (모바일): 수매 기간 중 내 지역 오늘 가격을 업데이트될 때마다 빠르게 보고 싶은 사람

**주요 연령층: 40대~70대 중장년·노년층**. 특히 시나리오 2는 대부분 모바일.

시나리오별 환경이 다름:
- **시나리오 2 (모바일)**: 핵심 정보(내 지역 시세)를 바로 보여주고, 추가 차트는 더보기/접힘 구조
- **시나리오 1 (PC)**: 풍부한 차트·테이블, 다중 필터, 비교 분석

이를 위한 설계:

- **내 지역 설정**: 첫 방문 시 지역 선택 → localStorage/zustand persist로 기억
- **시즌 중 대시보드 최상단**: 내 지역 오늘 등급별 시세 + 마지막 업데이트 시간
- **업데이트 표시**: 새 데이터 반영 시 "10:30 업데이트됨" 등 시각적 표시
- **다른 지역과 빠른 비교**: 내 지역 기준으로 타 지역 시세 나란히 비교 가능

### 시즌 기반 대시보드

송이버섯 수매는 9~11월에만 이뤄지므로, 대시보드를 시기에 따라 자동 전환:

- **시즌 중**: 실시간 KPI — 내 지역 시세, 전일 대비 변동, 등급별 현황
- **시즌 외**: 회고/분석 모드 — 직전 시즌 요약, 장기 트렌드, 시즌 패턴

같은 `/` 경로에서 시기에 따라 콘텐츠 자동 전환.

**시즌 판정 기준**: 데이터 유무 기반 — 최신 데이터가 7일 이내이면 시즌 중, 아니면 시즌 외.
고정 날짜(9~11월)보다 유연하고, 시즌이 일찍/늦게 시작·종료되어도 자동 대응.

### V1 기능 처리

V1의 기존 기능은 삭제하지 않고 V2에서 유사 기능으로 대체:

| V1 | V2 |
|---|---|
| Dashboard (MUI KPI + 바차트) | 시즌 중 대시보드 (내 지역 시세 + KPI + 차트) |
| DataAnalysis (필터 + 테이블 + 차트) | /analysis (연도별 비교 + 지역 비교 + 테이블) |
| RawData (주석 처리) | 삭제 — /analysis 상세 테이블로 흡수 |
| Help (주석 처리) | 삭제 |

## 페이지 구성

```
/ (Dashboard) — 시즌 여부에 따라 자동 전환
  시즌 중:
    ├── 내 지역 오늘 시세 (등급별, 업데이트 시간) ← 최상단
    ├── 전일 대비 변동 KPI
    ├── 시즌 가격 추이 (등급별 라인차트)
    └── 등급별 물량·가격 + 7일 변동

  시즌 외:
    ├── 직전 시즌 요약 리포트
    ├── 연간 거래량·금액 추이 (2013~현재)
    └── 시즌 패턴 분석 (월별 평균)

/analysis (데이터 분석)
  ├── 연도별 비교 (올해 vs 작년 오버레이)
  ├── 지역별 가격 비교 (랭킹)
  ├── 가격-물량 상관 분석
  └── 상세 테이블 (필터)
```

모바일 시즌 중 대시보드는 **내 지역 시세 + 전일 변동**이 뷰포트에 바로 보이고,
나머지 차트(시즌 추이, 등급별, 7일 변동)는 스크롤 아래 또는 접힘/더보기 구조.

## 디자인 시스템

### 원칙

1. **아름다운 UI가 최우선**: 접근성 때문에 디자인을 타협하지 않음
2. **충분한 여백**: 카드·섹션 간 넉넉한 간격으로 세련된 레이아웃
3. **정보 위계 명확**: 시세 숫자가 가장 크고, 나머지는 보조 역할
4. **색상 절제**: 의미 있는 곳에만 컬러. 장식적 색상 최소화
5. **디스플레이 모드 분리**: 기본(세련된 UI) + 큰글씨 모드(고연령 접근성)

### 디스플레이 모드

기본 UI는 아름답게 유지하고, 고연령 사용자를 위한 **큰글씨 모드**를 별도 제공.
zustand persist로 설정 기억. 헤더 우측에서 토글 (다크모드 토글 옆, 가/가 아이콘).

| 항목 | 기본 모드 | 큰글씨 모드 |
|---|---|---|
| 기본 폰트 | 0.875rem (14px) | 1.0625rem (17px) |
| 핵심 숫자(시세) | 2rem~2.5rem | 2.5rem~3rem |
| 카드 제목 | 0.875rem | 1.125rem |
| 캡션/보조 | 0.8125rem | 1rem |
| 최소 폰트 | 12px | 15px |
| 터치 타겟 | 44px | 52px |
| 차트 선 두께 | 2px | 3px |
| 차트 축 레이블 | 0.75rem | 0.9375rem |

구현: MUI 테마 변수 + `<html data-display="default|large">` 속성으로 전환. zustand persist로 상태 유지.

### 타이포그래피

- 시스템 폰트 스택 (Pretendard 또는 시스템 산세리프)
- 기본 모드 사이즈 스케일:
  - 핵심 숫자 (시세): 2rem~2.5rem, bold
  - 카드 제목: 0.875rem, medium, muted color
  - 본문/범례: 0.8125rem, regular
  - 캡션/업데이트 시간: 0.75rem, muted
- 숫자는 tabular-nums (고정폭 숫자)로 정렬감 확보

### 레이아웃

- 최대 너비: 1200px (데스크톱), 양쪽 패딩
- 카드 그리드: 모바일 1열 → 태블릿 2열 → 데스크톱 3~4열
- 카드 내부: padding 1.25~1.5rem, border-radius 0.75rem
- 차트 최소 높이: 모바일 240px, 데스크톱 320px
- 섹션 간 간격: 1.5~2rem

### D3 차트 스타일 가이드

- 축 레이블: muted color, 0.75rem — 데이터가 주인공, 축은 보조
- 그리드라인: 매우 연한 색 (opacity 0.1~0.15), 가로만 사용
- 툴팁: 반투명 배경 + backdrop-blur, 둥근 모서리, 그림자
- 애니메이션: 진입 시 fade-in + draw (0.6~0.8초), 과하지 않게
- 범례: 차트 상단 또는 우측, 인라인 배치로 공간 절약
- 반응형: SVG viewBox 기반, 모바일에서 범례 하단 이동
- 큰글씨 모드: 선 두께·포인트 반경·축 레이블 사이즈 CSS 변수로 스케일업

### 인터랙션

- 호버 (PC): 미세한 elevation 변화 (그림자 증가) — 배경색 변경 아닌 그림자로
- 가격 변동 표시: 숫자 앞에 ▲/▼ + 컬러 (상승=앰버, 하락=틸). 변동 없으면 — (회색)
- 로딩: 스켈레톤 UI (shimmer 효과), 레이아웃 시프트 방지
- 전환: 페이지/섹션 전환 시 부드러운 fade (150~200ms)

## 구현 단계

MUI v7 위에서 점진적으로 기능 추가. V1 코드를 최대한 활용.
v2 브랜치에서 작업, 완성 후 main에 머지.

### Phase 0: 테마 + 인프라

- MUI 테마를 Modern Forest 팔레트로 교체 (`theme.ts`)
- 큰글씨 모드 테마 변수 추가 + 토글 UI (헤더 우측)
- zustand 스토어 도입 (지역 설정, 디스플레이 모드 — persist)
- 내 지역 설정 UI (온보딩 모달 + 헤더 셀렉트)

### Phase 1: 데이터 파이프라인 확장

- `generate-stats.ts` 확장:
  - 전일 대비 등급별 가격 변동률
  - 지역별 등급별 일일 시세
  - 지역별 평균 가격 랭킹
  - 연도별 동기 비교 데이터
  - 시즌별 월간 평균 (가격·물량)
  - 시즌 요약 (총량, 평균, 최고/최저)
- 새 manifest 파일 생성 또는 기존 구조 확장
- Zod 스키마 추가

### Phase 2: 시즌 중 대시보드 (핵심)

- 기존 Dashboard.tsx를 시즌 중 모드로 리팩토링

### Phase 3: 시즌 외 대시보드

- 시즌 판정 로직 (최신 데이터 7일 기준)
- 같은 Dashboard.tsx에서 시즌 외 콘텐츠 자동 전환

## 오프라인/저속 네트워크 대응

산지에서 모바일 사용 시 통신 불안정을 고려:

- IndexedDB에 마지막 로드 데이터 캐시 유지 (기존 Dexie 구조 활용)
- 오프라인/로드 실패 시 캐시 데이터 표시 + "오프라인 — 마지막 업데이트: 10/15 10:30" 배너
- 네트워크 복구 시 자동 갱신

## 데이터 수집 파이프라인

- GitHub Actions cron으로 공판 데이터 자동 수집 (기존 V1 파이프라인 유지)
- V2에서는 `generate-stats.ts` 확장만 필요, 수집 로직 자체는 변경 없음

## 현재 데이터 역량

- 2013~2025년 데이터 보유
- 레코드당 44개 수치 필드 (6개 등급 × 수량·단가 + 총 수량·금액)
- 지역/조합별 분류
- IndexedDB 복합 인덱스: [date+region], [date+union], [date+region+union]
- 사전 집계: daily/weekly/yearly manifest JSON
