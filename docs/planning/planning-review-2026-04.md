# 기획 검토 (2026년 4월)

V2 구현 이후 코드베이스 전체를 분석하여 현재 기능 완성도, 신규 기능 방향, 사용자 시나리오별 문제점을 정리한 문서입니다.

---

## 1. 현재 기능의 완성도

### 잘 되어 있는 부분

- 시즌/비시즌 자동 전환 로직이 깔끔함
- 지역 선택 → 조합 연동 필터링 잘 동작
- D3 차트 반응형 처리 (ResizeObserver 기반)
- IndexedDB 캐싱 + 3단계 폴백 버전 체크
- useMemo 기반 파생 데이터 최적화
- 데이터 분석 페이지의 다차원 필터 + 비교 모드
- 비시즌 대시보드 (시즌 요약, 연도별 추이, 월별 패턴, 지역 순위)

### 미완성 / 개선 필요

| 영역 | 현황 | 개선 포인트 |
|------|------|------------|
| 에러 처리 | 네트워크 에러가 `console.error`만 출력 | 사용자에게 보이는 에러 UI 필요 (토스트, 에러 바운더리) |
| 큰 글씨 모드 | `displayMode: "large"` 상태만 존재, CSS 미구현 | 실제 폰트 스케일링 CSS 연결 필요 |
| 온보딩 | 닫기/건너뛰기 불가, 지역 선택 강제 | "나중에 설정" 옵션 추가 필요 |
| 데이터 신선도 | "마지막 업데이트" 시점 표시 없음 | 기준 날짜/시간 표시 필요 |
| 새로고침 | 수동 데이터 갱신 기능 없음 | 리프레시 버튼 필요 (시즌 중 특히) |
| 빈 데이터 | 대시보드에서 `regionGradeBreakdown` 비었을 때 빈 그리드만 표시 | 의미 있는 빈 상태 메시지 필요 |

## 2. 신규 기능 방향

### 높은 가치 (사용자 임팩트 큼)

#### 알림/구독 기능

- "내 지역 시세가 X원 이상/이하일 때 알림" 같은 조건부 알림
- 현재 데이터 파이프라인(GitHub Actions → Vercel)은 이미 자동화되어 있어서, 클라이언트 측 비교 로직만 추가하면 브라우저 Push 알림 가능
- 송이 거래자 입장에서 가장 실용적인 기능

#### 비교 프리셋

- 현재 비교 모드는 날짜를 수동 입력해야 함
- "전년 동기", "지난주", "시즌 시작 대비" 같은 원클릭 프리셋 추가
- 분석 페이지의 사용 허들을 크게 낮춤

#### 데이터 내보내기

- 차트를 PNG/SVG로, 테이블을 CSV로 다운로드
- 조합 관계자가 보고서에 활용할 수 있음

### 중간 가치

#### 가격 예측/트렌드 표시

- 이동 평균선, 전년 대비 추세선
- 현재 산점도에 회귀선이 있으므로 확장 가능

#### 조합 간 비교 뷰

- 현재는 지역 비교만 존재
- 같은 지역 내 조합끼리 비교하는 니즈가 있을 것

#### 즐겨찾기 / 커스텀 대시보드

- 자주 보는 필터 조합을 저장
- Zustand 기반이라 persist 확장 쉬움

### 낮은 가치 (있으면 좋지만 급하지 않음)

- 다국어 지원 (현재 한국어 전용, 타겟 사용자 고려 시 불필요할 수 있음)
- 차트 접근성 강화 (ARIA, 키보드 내비게이션)
- 테이블 가상 스크롤링 (데이터 규모가 극단적이지 않으면 체감 낮음)

## 3. 사용자 시나리오별 문제점

### 시나리오 A: 송이 채취자가 오늘 시세를 확인하러 옴

```
첫 방문 → 온보딩 강제 팝업 → 지역 선택해야만 진행 가능
                                 ⚠️ "일단 빨리 시세만 보고 싶은데" 좌절
→ 지역 선택 완료 → 대시보드 로딩 → 데이터 다운로드 (첫 방문이라 전체)
                                     ⚠️ 로딩 중 진행률 표시 없음
→ 시세 확인 → 다른 지역도 보고 싶음
              ⚠️ 조합 단위로 바로 비교하는 뷰가 없음
```

**핵심 문제**: 첫 방문 허들이 높고, 로딩 피드백이 부족함

### 시나리오 B: 조합 관계자가 시즌 분석 보고서를 작성

```
데이터 분석 페이지 진입 → 필터 설정 (지역, 조합, 기간)
→ 전년 대비 비교하고 싶음 → 비교 모드 토글 ON
                            ⚠️ 비교 기간을 수동으로 직접 입력해야 함
                            ⚠️ "전년 동기" 같은 프리셋이 없음
→ 차트/KPI 확인 → 보고서에 넣고 싶음
                   ⚠️ 차트 다운로드/복사 기능 없음
                   ⚠️ 테이블 CSV 내보내기 없음
→ 결국 스크린샷 찍어서 사용
```

**핵심 문제**: 분석 결과물을 외부로 가져가는 수단이 없음

### 시나리오 C: 시즌이 아닌 시기에 방문

```
대시보드 진입 → 비시즌 모드 자동 전환
→ 과거 시즌 요약 표시 (KPI, 연도별 추이, 월별 패턴, 지역 순위)
  ✅ 이 부분은 잘 되어 있음
→ "올해 시즌은 언제 시작하나?" 같은 정보는 없음
  ⚠️ 시즌 카운트다운이나 예상 시작 안내가 없음
→ 데이터 분석 페이지에서 과거 여러 시즌 비교 가능
  ⚠️ 시즌 전체 트렌드 (5년간 평균 단가 추이)를
     한눈에 보는 뷰가 별도로 없음
```

**핵심 문제**: 비시즌 시 사용자 재방문 동기가 약함

### 시나리오 D: 모바일에서 빠르게 확인

```
모바일 브라우저 → 대시보드 → 차트가 세로로 쌓여서 스크롤이 김
                              ⚠️ 가장 중요한 정보(오늘 시세)까지
                                 도달하는 데 스크롤 많이 필요할 수 있음
→ 7일 추이 차트 → 범례가 아래로 내려가서 차트 영역 축소
                   ⚠️ 6개 등급 범례가 모바일에서 2~3줄 차지
→ 데이터 분석 → 필터 UI가 모바일에서 조작하기 번거로움
                 ⚠️ 다중 Select + DatePicker가
                    작은 화면에서 불편할 수 있음
```

**핵심 문제**: 모바일 정보 밀도 최적화 필요 (v2-plan.md의 모바일 퍼스트 방향과 연결)

## 4. 종합 우선순위

| 순위 | 항목 | 근거 | 난이도 |
|------|------|------|--------|
| 1 | 온보딩 "건너뛰기" + 로딩 프로그레스 | 첫 방문 이탈 방지 | 낮음 |
| 2 | 에러 UI + 데이터 갱신 버튼 | 기본 품질 보장 | 낮음 |
| 3 | "마지막 업데이트" 표시 | 데이터 신뢰도 | 낮음 |
| 4 | 비교 프리셋 (전년 동기 등) | 분석 사용성 | 중간 |
| 5 | 차트/테이블 내보내기 | 실용 가치 | 중간 |
| 6 | 모바일 정보 우선순위 조정 | 접근성 | 중간 |
| 7 | 알림 기능 | 재방문/리텐션 | 높음 |
