## 1. 제품 개요

### 정체성

**송이버섯 공판 시세 대시보드** — 전국 조합별 일별 경매 시세를 시각화하는 SPA.
2013년부터 현재까지의 데이터를 IndexedDB에 로컬 캐싱하여 오프라인·저속 환경에서도 동작한다.

### 핵심 기술 스택

React 19 + MUI v7 + D3.js / Vite 7 / Dexie (IndexedDB) / Zustand / Vercel 배포

### 주요 사용자 2종

| 구분                   | 환경                   | 핵심 니즈                       |
| ---------------------- | ---------------------- | ------------------------------- |
| **채취자·농가**        | 모바일, 산지·시장 현장 | 내 지역 오늘 시세를 빠르게 확인 |
| **조합 관계자·분석가** | PC                     | 시즌 통계 분석, 보고서 작성     |

**주요 연령층: 40~70대** — 가독성과 접근성이 UX의 핵심 변수.

## 3. 사용자 시나리오별 현재 문제

### 시나리오 A — 채취자가 오늘 시세 확인 (모바일)

```
앱 진입 → 첫 방문 시 온보딩 (✅ 건너뛰기 가능)
→ 대시보드 로딩 (✅ 진행률 표시)
→ 시세 확인 ← ⚠️ KPI 카드까지 스크롤 필요 (375px 기준)
→ 다른 지역 비교 ← ⚠️ 조합 단위 빠른 비교 뷰 없음
→ 글자가 작아서 불편 ← ⚠️ 큰글씨 모드 토글은 있지만 CSS 미연결
```

**핵심 문제**: 모바일에서 핵심 정보(오늘 시세)까지 도달 거리가 멂 + 글자 크기 미적용

### 시나리오 B — 조합 관계자의 보고서 작성 (PC)

```
분석 페이지 → 필터 설정 → 비교 모드 ON
→ 비교 기간 설정 ← ⚠️ 수동 입력 필요 ("전년 동기" 버튼 없음)
→ 차트·KPI 확인 → 보고서 삽입
← ⚠️ 차트 다운로드, 테이블 CSV 내보내기 없음
→ 결국 스크린샷 사용
```

**핵심 문제**: 분석 결과물을 외부로 가져가는 수단 없음

### 시나리오 C — 비시즌 방문

```
대시보드 → 비시즌 모드 (✅ 요약/추이/패턴/순위 잘 구현됨)
→ "올해 시즌은 언제?" ← ✅ 카운트다운 카드 표시
→ 재방문 동기 없음 ← ⚠️ 시즌 시작 알림 없음
```

## 5. Phase별 구현 계획

### Phase A — 큰글씨 모드 CSS 연결 ✅ 완료 (2026-04-18)

**목표**: `displayMode: "large"` 토글 시 모든 화면에서 즉시 시각적 변화 발생

**구현 방법**

1. `App.tsx` 또는 `main.tsx`에서 `displayMode` 상태를 `<html data-display="...">` 속성으로 동기화
2. `src/App.css`에 CSS 변수 추가:

```css
:root {
  --font-base: 0.875rem;
  --font-heading: 0.875rem;
  --font-kpi: 2rem;
  --font-caption: 0.75rem;
  --touch-target: 44px;
}
[data-display="large"] {
  --font-base: 1.0625rem;
  --font-heading: 1.125rem;
  --font-kpi: 2.75rem;
  --font-caption: 1rem;
  --touch-target: 52px;
}
```

3. MUI 테마 컴포넌트 오버라이드 또는 `sx` 시스템으로 CSS 변수 참조
4. D3 차트: `chartMargins.ts`, `legendLayout.ts`에서 `isMobile` 외 `isLargeFont` 조건 추가

**검증**: 큰글씨 모드 토글 → KPI 숫자·카드 제목·캡션 모두 커짐. 차트 축 레이블도 증가.

---

### Phase B — 비교 프리셋 ✅ 완료 (2026-04-18)

**목표**: 원클릭으로 자주 쓰는 비교 기간 설정

**구현 방법**

1. `src/utils/analysis/dateRange.ts`에 프리셋 함수 추가:
   - `getPresetLastWeek(baseRange)` → 현재 기간 대비 1주 전
   - `getPresetLastYear(baseRange)` → 전년 동기
   - `getPresetSeasonStart(baseRange)` → 시즌 시작 대비

2. `src/components/DataAnalysis/Filters/ComparisonToggle.tsx`에 칩 버튼 UI 추가:

   ```
   [비교 모드 ON] 일 때:
   ┌─────────────────────────────────────┐
   │ 프리셋: [전년 동기] [지난주] [시즌 시작] │
   │ 또는 직접 입력: [시작일] ~ [종료일]    │
   └─────────────────────────────────────┘
   ```

3. 프리셋 선택 시 비교 날짜 범위 자동 설정 (`onComparisonRangeChange` 호출)

**검증**: "전년 동기" 클릭 → 비교 기간이 1년 전 같은 날짜로 자동 설정됨.

---

### Phase C — 차트/테이블 내보내기

**목표**: 분석 결과물을 파일로 저장

**구현 방법**

1. **차트 PNG 내보내기**
   - SVG 요소 직렬화 → `<canvas>`에 그리기 → `toBlob()` → 다운로드
   - 공통 훅 `src/hooks/useChartExport.ts` 작성
   - 각 차트 컴포넌트 우측 상단에 다운로드 아이콘 버튼 추가 (호버 시 노출)

2. **테이블 CSV 내보내기**
   - `src/utils/exportCsv.ts` — 현재 필터 결과 배열 → CSV 문자열 → Blob
   - `TableSection.tsx`에 "CSV 다운로드" 버튼 추가

**검증**: 차트 다운로드 버튼 클릭 → PNG 파일 저장. 테이블 버튼 → CSV 파일 저장 (한글 인코딩 확인).

### Phase D — 모바일 정보 밀도 최적화 ✅ 완료 (2026-04-18)

**목표**: 375px에서 KPI가 스크롤 없이 보임

**구현 방법**

1. **시즌 중 대시보드 레이아웃 조정**
   - `DashboardHeader.tsx`: 타이틀 + 컨트롤을 한 줄로 압축 (모바일)
   - `DashboardKpiRow.tsx`: 4개 → 2×2 그리드, 카드 패딩 축소
   - 차트 섹션은 KPI 아래로 내림

2. **범례 최적화**
   - `DashboardChartWeeklyToggle`: 모바일에서 범례 텍스트 크기 축소, 아이콘만 표시 옵션
   - `legendLayout.ts`의 wrap 로직 재검토

3. **분석 필터 모바일 UX**
   - `AnalysisFilters.tsx`: 모바일에서 접이식 패널 (Collapse + "필터 펼치기" 버튼)

**검증**: 375px Chrome DevTools에서 KPI 4개 전체가 스크롤 없이 화면 안에 보임.

## 8. 기술 부채 및 주의사항

- **D3 차트는 직접 구현 유지** — Recharts/Chart.js 전환 제안 금지
- **큰글씨 모드와 다크모드는 독립** — 조합 4가지 모두 QA 필요
- **CSV 인코딩**: 한글 포함 시 UTF-8 BOM 필요 (Excel 호환)
- **PNG 내보내기**: cross-origin 이미지 없으므로 canvas taint 문제 없음
